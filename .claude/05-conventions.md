# 05 — Coding Conventions

---

## API Routes

### Response Shape
Every API route MUST return this consistent shape. No exceptions.

```typescript
// Success
{ success: true,  data: T }

// Error
{ success: false, error: string }
```

### Validation
Always validate request body with Zod before touching the database.

```typescript
const schema = z.object({ ... });
const parsed = schema.safeParse(body);
if (!parsed.success) {
  return NextResponse.json({ success: false, error: "Invalid input" }, { status: 400 });
}
```

### Auth Check in Protected API Routes
```typescript
import { getAdminFromCookie } from "@/lib/auth/admin";

const admin = await getAdminFromCookie();
if (!admin) {
  return NextResponse.json({ success: false, error: "Unauthorized" }, { status: 401 });
}
```

### Route File Location
```
Public APIs:     src/app/api/[resource]/route.ts
Admin APIs:      src/app/api/admin/[resource]/route.ts
Auth APIs:       src/app/api/auth/admin/login/route.ts
```

## Database

### DB Calls — Server Only
Database calls go ONLY in:
- Server Components
- API Route Handlers (`route.ts`)
- Server Actions

NEVER in client components (`"use client"`).

### Drizzle Client
```typescript
import { db } from "@/lib/db";
import { products } from "@/lib/db/schema";
```

### Schema Changes
```
1. Edit file in src/lib/db/schema/
2. npm run db:push:local
3. Never edit files in drizzle/migrations/ manually
```

### Relations
Both sides of every relation must be declared.
```typescript
// In products.ts
export const productsRelations = relations(products, ({ many }) => ({
  images: many(productImages),   // parent side
}));

// In products.ts — productImages section
export const productImagesRelations = relations(productImages, ({ one }) => ({
  product: one(products, { ... }) // child side — REQUIRED
}));
```

## Pricing

- All prices stored as **integers in paise** (₹1 = 100 paise)
- ₹999 is stored as `99900`
- Always use `formatPrice()` helper for display

```typescript
// src/lib/utils/helpers.ts
formatPrice(99900) // → "₹999"
formatPrice(150000) // → "₹1,500"
```

Never store decimal prices. No floats in DB.

## Slugs

Auto-generate slugs from names using `slugify()` helper.

```typescript
slugify("Intel Core i5 Motherboard") // → "intel-core-i5-motherboard"
```

Slugs must be unique. If a slug already exists, append `-2`, `-3`, etc.

---

## SKUs

Format: `VP-[CATEGORY]-[RANDOM]`
Examples: `VP-LAP-A3K9`, `VP-MB-Z2X1`
Admin can set manually or auto-generate.
Must be unique across all variants.

---

## Order Numbers

Format: `VP-ORD-YYYYMMDD-NNN`
Example: `VP-ORD-20240318-001`
NNN is a sequential counter per day.
Generated by `generateOrderNumber()` in `src/lib/utils/helpers.ts`.

---

## Internal Tracking Codes

Format: `VP-XXXXXX` (6 char nanoid, uppercase alphanumeric)
Example: `VP-A3KZ9M`
Generated at order creation by `generateInternalTrackingCode()`.

## File & Component Naming

```
page.tsx            Next.js page (default export)
layout.tsx          Next.js layout (default export)
route.ts            API route handler
ComponentName.tsx   React component (PascalCase)
useHookName.ts      React hook (camelCase with "use" prefix)
helperName.ts       Utility function file (camelCase)
```

## TypeScript

- Strict mode is on — no `any` unless absolutely unavoidable
- Use Drizzle inferred types as the source of truth:
  ```typescript
  type Product    = typeof products.$inferSelect;
  type NewProduct = typeof products.$inferInsert;
  ```
- Shared types go in `src/types/index.ts`
- API response wrapper type:
  ```typescript
  type ApiResponse<T> = { success: true; data: T } | { success: false; error: string };
  ```

---

## Images

- Always upload to Supabase Storage bucket: `product-images`
- Store the public URL in `product_images.url`
- Use `NEXT_PUBLIC_SUPABASE_URL` + storage path to construct URLs
- `next/image` component must be used for all product images
- `next.config.ts` already has Supabase hostname whitelisted

## Tailwind & Styling

- Use the `cn()` utility (clsx + tailwind-merge) for conditional classes
- No inline styles
- Dark mode: not planned yet — keep styles light mode only for now

---

## Error Handling

- API routes: always wrap in try/catch, return `{ success: false, error }` on failure
- Log server errors with `console.error()` — never expose raw error messages to client
- Client-side: use `sonner` toast for user-facing error messages

---

## Import Aliases

Use `@/` alias for all internal imports (configured in tsconfig.json):
```typescript
import { db } from "@/lib/db";
import { Button } from "@/components/ui/button";
import { formatPrice } from "@/lib/utils/helpers";
```
